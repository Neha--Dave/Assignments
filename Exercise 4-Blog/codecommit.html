Code commit to git

Git is a free & open source, distributed version control system designed with complete history and full revision tracking capabilities, not dependent on network access or a central server. It has a concept of staging area where your code is placed before committing. this means that you first have a code on your local system. It then goes to the staging area and then on the server. Each of these have a different command to be fired. Some find these steps confusing. They are explained in a simple manner as follows:


» git add: Ensures that git is introduced to the file which will be later be added to the repository.

» git commit: Places the file in the staging area

» git push: Pushes the file in the repository
